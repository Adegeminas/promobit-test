ТЕСТОВОЕ ЗАДАНИЕ: Volume Manager
=================================

Стек: Vue 3 + TypeScript + Composition API + <script setup>
Срок: 2 дня с момента получения


КОНТЕКСТ

Реализация формы создания нового Volume (тома) для админ-панели
системы управления хранением данных.

Volume отправляется на бэкенд в виде:

    {
        name:      string
        type:      "block" | "file"
        sizeBytes: number
    }


ЧТО НУЖНО СДЕЛАТЬ

Форма создания Volume. При отправке — вывести рядом с формой
итоговый объект, который пойдёт на бэкенд.


1. ПОЛЕ name

    - Обязательное
    - Только латиница, цифры и дефис
    - Имя должно быть уникальным — проверяй через mock API (описан ниже)


2. ПОЛЕ type

    - Выбор из "block" / "file"


3. ПОЛЕ size

    - Пользователь вводит число и выбирает единицу измерения
    - При отправке конвертируется в байты (поле sizeBytes)
    - Поле должно быть удобным для пользователя


4. ПЕРЕКЛЮЧАТЕЛЬ СИСТЕМЫ ИЗМЕРЕНИЯ

В шапку приложения переключатель:

    Десятичная: KB (x1000),  MB (x1000^2),  GB (x1000^3)
    Двоичная:   KiB (x1024), MiB (x1024^2), GiB (x1024^3)

Переключатель влияет на поле size.


MOCK API

В папке с заданием лежит файл mock.js.
Копируем в свой проект и подключаем напрямую — HTTP-сервер не нужен.

Доступные методы:

    mockApi.checkName(name)
        Принимает строку
        Возвращает Promise<{ unique: boolean }>
        Имитирует задержку и случайные ошибки


РЕЗУЛЬТАТ ОТПРАВКИ

При нажатии "Сохранить" показать рядом с формой:

    {
        "name": "...",
        "type": "...",
        "sizeBytes": ...
    }


ТРЕБОВАНИЯ

    - Vue 3 + TypeScript + Composition API + <script setup> — обязательно
    - Тесты для того, что считаете важным
